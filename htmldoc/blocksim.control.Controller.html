class blocksim.control.Controller.AntiWindupPIDController(name: str, shape_estimation: tuple, snames: Iterable[str], coeffs: float = (0, 0, 0, 0, 0, 0))[source]

Bases : blocksim.control.Controller.AController

One-dimensional PID controller with anti-windup

The inputs of the computer are estimation and setpoint The outputs of the computer are command and integral

The estimation \(\hat{X}\) must contain the state you want to control \(X\) and its derivative \(\dot{X}\).:

\(\hat{X} = (X, \dot{X}, ...)^T\)

The parameters P ,I and D are to be defined by the user :

  • P : Proportinnal gain

  • I : Integral gain

  • D : Derivative gain

  • Umin : if the command u is < Umin, then u = Umin

  • Umax : if the command u is > Umax, then u = Umax

  • Ks : gain of the anti-windup effect

Paramètres

  • name – Name of the element

  • shape_estimation – Shape of the data expected by the estimation (> 2)

  • coeffs – Coefficients of the retroaction (P, I, D, Umin, Umax, Ks)

class blocksim.control.Controller.LQRegulator(name: str, shape_setpoint: tuple, shape_estimation: tuple, snames: Iterable[str])[source]

Bases : blocksim.control.Controller.AController

LQ regulator

The inputs of the element are estimation and setpoint The outputs of the computer are command The size of the setpoint vector must be the same as the command vector

The parameters Q and R are to be defined by the user :

  • Q : State weight matrices

  • R : Input weight matrices

The parameters K, S, E and N are computed thanks to the method LQRegulator.computeGain

  • K : State feedback gains

  • S : Solution to Riccati equation

  • E : Eigenvalues of the closed loop system

  • N : Precompensation gain

Paramètres

  • name – Name of the element

  • shape_setpoint – Shape of the setpoint data

  • shape_estimation – Shape of the data expected by the estimation (> 2)

computeGain(precomp: bool = True)[source]

Computes the optimal gain K, and the correct precompensation gain N

K minimizes the quadratic cost \(J = \int_0^\infty (x' Q x + u' R u + 2 x' N u) dt\)

and N is such that the steady-state error is 0

Needs to have the following parameters set:

  • matA (n x n) State (or system) matrix

  • matB (n x m) Input matrix

  • matC (p x n) Output matrix

  • matD (p x m) Feedthrough (or feedforward) matrix

  • matQ covariance matrix of model noise

  • matR covariance matrix of measurement noise

At the end of the call, the following attributes are updated :

  • matK : optimal gain

  • matS : solution to Riccati equation

  • matE : eigenvalues of the closed loop system

  • matN : precompensation gain

Paramètres

precomp – If True, also computes the N matrix such as for a setpoint c, the command u = N.c - K.x suppresses the steady-state error.
